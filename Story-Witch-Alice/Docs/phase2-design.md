# 故事魔女爱丽丝 —— Phase 2 设计思路汇总

---

## 一、从 MVP 到 Phase 2

MVP 阶段我们跑通了核心循环：**拖拽元素 → 丢进锅里 → 合成新元素 → 合成树记录**。这是一个"能跑通"的原型。

但 MVP 有个致命问题：**缺乏留存动力**。玩家合成几次后，新鲜感消退，就再也不会打开游戏了。我们花掉的 Token 成本也无法回本。

所以 Phase 2 的核心目标是：**让玩家每天都想回来看看**。

---

## 二、核心设计转变：从"一次性故事"到"持续世界模拟"

### 2.1 游戏的核心循环

```
玩家合成爱丽丝 → 派遣到世界（影响世界属性）→ 下线
→ AI 离线推演世界发展 → 玩家上线（收取事件卡片，看到世界变化）
→ 用奖励继续合成新爱丽丝 → 循环
```

### 2.2 正反馈的三层结构

借鉴《妹相随》的"数字反馈"设计——信赖+2、性趣+1，玩家每次操作都能看到直观的数字变化，产生"我在推动这个世界"的实感。

| 层级 | 时机 | 反馈内容 |
|------|------|---------|
| **即时反馈** | 合成完成瞬间 | 新爱丽丝诞生 + 属性标签 + 世界属性变化数字弹出 |
| **中程反馈** | 派遣爱丽丝后 | 世界雷达图变化（混沌性+3，文明度+1） |
| **长期反馈** | 每次上线 | 事件卡片："你下线期间，植物学家爱丽丝在废墟中培育出了第一株变异小麦" |

## 三、爱丽丝的统一设计哲学：一个爱丽丝，三重身份

### 3.1 核心原则

在最初的讨论中，我们尝试把爱丽丝分成"身份爱丽丝"和"功能性爱丽丝"。但经过深入思考，我们意识到这是一个错误的二分法。

**每个爱丽丝都应该是一个完整的人。** 她不应该只是"工具"或"故事摆设"，而是同时具备三重身份：

| 维度 | 说明 |
|:---|:---|
| **职业** | 她的身份标签（植物学家、岩石学家、哲学家） |
| **功能** | 她能为世界做什么（发现新元素、培育植物、细分矿物） |
| **结局** | 她的故事收束（在这个世界的某个角落，发生了只属于她的故事） |

合成不是"制造工具"，而是**赋予爱丽丝一段人生**。

### 3.2 催化剂：从空白灵魂到有热情的人

植物 + 爱丽丝，为什么就能变成植物学家？中间缺少了一个关键的催化剂——**让她对这个领域产生"执着"的东西。**

这个催化剂可以是：
- **生命力**：赋予她探索生命奥秘的热情
- **时间**：赋予她对永恒的思考与耐心
- **地图**：赋予她对未知的探索欲
- **扳手**：赋予她动手改造世界的意愿
- **风**：赋予她对自由与变化的追求
- **水**：赋予她包容与滋养的温柔
- **火**：赋予她突破与冒险的激情

所以合成公式统一为：

```
核心元素 + 爱丽丝 + 催化剂 = 职业爱丽丝
```

这解决了"为什么元素加爱丽丝就能变成某种职业"的逻辑问题。催化剂是爱丽丝从"空白灵魂"转变为"有热情的人"的关键。而生命力本身是有限的（100点），玩家需要谨慎分配。

### 3.3 当前所有爱丽丝（统一模板）

| 爱丽丝 | 合成公式 | 职业 | 功能（对世界的影响） | 可能的结局类型 |
|:---|:---|:---|:---|:---|
| **岩石学家爱丽丝** | 石 + 爱丽丝 + 地图 | 地质学者 | 投入世界后，发现新矿物（金属元素解锁） | 在某个远古地层中发现稀有矿石；或在火山喷发中失踪 |
| **金属学家爱丽丝** | 金属 + 爱丽丝 + 扳手 | 工匠/冶金师 | 将"金属"细分为铁、铜、锡等 | 她锻造出一把传世之剑；或她的工坊被时代遗忘 |
| **生命爱丽丝** | 生命力 + 爱丽丝 + 时间 | 生命探索者 | 穿梭世界切片，寻找生命痕迹（蓝藻等） | 她在原始海洋中发现了第一个细胞；或她目睹了一个物种的灭绝 |
| **植物学家爱丽丝** | 植物 + 爱丽丝 + 生命力 | 植物学者 | 培育新作物，发现可食用植物 | 培育出核纪元第一株可食用小麦（救世主）；或她的花园被荒漠吞没 |
| **天气学家爱丽丝** | 雨 + 爱丽丝 + 风 | 气象观测者 | 预测天气，影响世界气候属性 | 她预言了一场改变文明走向的暴风雨；或她追逐龙卷风再未归来 |
| **泥土爱丽丝** | 土 + 爱丽丝 + 水 | 沉默的守望者 | 被动滋养大地，缓慢提升世界生命力 | 她只是一捧泥土，不在乎恐龙的奔跑与陨石的坠落 |
| **哲学家爱丽丝** | 哲学 + 爱丽丝 + 时间 | 思想者 | 影响世界秩序性和文明度 | 她的思想在千年后被人发现，成为新文明的基石 |
| **科学家爱丽丝** | 科学 + 爱丽丝 + 火 | 实验者 | 推动技术进步，降低世界神秘度 | 她的实验改变了世界；或她的实验室发生了一场大爆炸 |

### 3.4 以后设计新爱丽丝的模板

只需回答三个问题：
1. **她是谁？**（职业/身份标签）
2. **她能为世界做什么？**（功能——解锁新元素、细分已知元素、改变世界属性、触发特定事件等）
3. **她的故事会怎么收场？**（结局——基于她的属性和世界状态，在六种结局光谱中落点）

## 四、爱丽丝的生命力系统

- 爱丽丝拥有 100 点生命力
- 每次与其他元素合成，消耗一定比例（例如 10%）
- 每日休息（或使用道具）可恢复生命力
- 生命力本身也可以作为合成元素：生命力 + 爱丽丝 + 时间 = 生命爱丽丝
- 生命力耗尽 → 爱丽丝进入虚弱状态，无法合成

这个系统让生命力既是资源，也是催化剂，玩家需要在"消耗生命力合成"和"保留生命力恢复"之间做出选择。

## 五、世界切片与开拓者

### 5.1 世界切片

世界切片是"不同时间和地点的快照"，是爱丽丝探索和结局发生的舞台：

- **古宙纪·原始海洋**：生命爱丽丝在这里发现蓝藻
- **远古纪·原始森林**：泥土爱丽丝的结局舞台
- **核纪元·废墟都市**：植物学家爱丽丝培育变异小麦的地方

开拓者（商人角色）带回世界切片坐标，玩家派遣功能性爱丽丝去探索。

### 5.2 开拓者

- 在宇宙中游荡的流浪商人
- 带回世界切片坐标
- 用灵感货币与玩家交易稀有元素

## 六、六维元属性体系

所有元素都拥有统一的六维属性，从 -5 到 +5：

| 属性 | 名称 | 正极含义 | 负极含义 | 对应世界观 |
|:---|:---|:---|:---|:---|
| `order` | 秩序性 | 稳定、规则、可控 | 混沌、随机、不可预测 | 宇宙结构 |
| `creativity` | 创造性 | 创新、变革、启发 | 保守、重复、僵化 | 文明科技 |
| `empathy` | 共情性 | 连接、治愈、利他 | 冷漠、隔离、自私 | 社会纽带 |
| `desire` | 欲望性 | 追求、执着、行动 | 被动、消解、献祭 | 个体意志 |
| `mystery` | 神秘度 | 未知、魔法、灵性 | 已知、科学、平凡 | 超自然程度 |
| `vitality` | 生命力 | 生长、活力、繁荣 | 衰败、死亡、荒芜 | 生态系统 |

### 属性与恶的关系

我们认为**几乎所有的"恶"都源于"欲望"**。高欲望 + 低共情 = 掠夺者；高欲望 + 高共情 = 改革者。欲望本身不是道德标签，而是一种存在方式——高欲望的东西不一定"恶"，但它更容易引发冲突。

### 物理属性 vs 元属性

- **物理属性**：不统一，每个元素填自己相关的就行（水填湿度，金属填硬度）。它们是**合成条件**。
- **元属性**：统一用六维度，所有元素都填。它们是**产物性格和世界影响的燃料**。
- 两者混在同一个 `Properties` 列表中，用 key 区分。

## 七、结局判定机制

爱丽丝被投入世界后，结局不是随机的，也不只有"世界毁灭"，而是基于**匹配度 + 影响力**的多元叙事判定。

### 判定流程

1. **计算影响力**：取爱丽丝六维中的最高值作为核心驱动力，结合其他正属性数量计算影响力
2. **匹配度判定**：将爱丽丝属性与世界当前最匮乏的属性做比较，决定结局基调
3. **世界切片背景**：时代和地点为结局提供具体舞台

### 结局类型光谱

| 结局类型 | 判定条件 | 情绪基调 |
|:---|:---|:---|
| **救世主结局** | 高匹配 + 高影响力 + 积极属性 | 希望、温暖、史诗 |
| **殉道者结局** | 高匹配 + 中影响力 + 牺牲倾向 | 崇高、悲伤、感动 |
| **旁观者结局** | 低匹配 + 低影响力 | 疏离、宁静、诗意 |
| **异化者结局** | 冲突匹配 + 高影响力 | 诡异、颠覆、黑暗 |
| **齿轮结局** | 中匹配 + 低影响力 | 平凡、重要、纪实 |

### 泥土爱丽丝结局示例

> 天空再次被陨石撕裂了一道口子，一只窃蛋龙从爱丽丝身上跑过。风儿似乎察觉到了什么，携带着一股焦虑穿梭在这片森林。哦，一群小型啮齿动物找到了落在爱丽丝身上的浆果，真是走运。阳光刺穿天空被撕裂的伤口，又轻柔透过树荫，把少数的光照在这片温馨的土地上。土地的震动似乎预示着什么，但是爱丽丝只是一捧泥土，她不在乎。

这是一个典型的**低匹配 + 旁观者结局**：世界不需要她的温和秩序，她也没有力量去改变，只能安静地见证一切。

## 八、灵感货币与续写机制

- **灵感货币**：有限资源，从开拓者交易、世界事件奖励中获取
- **续写结局**：消耗货币，让 AI 为一个已有结局补充更多细节、延伸支线、甚至改写走向
- **成本控制**：免费第一版结局是引子，续写是付费深度体验，同时控制 Token 成本

## 十、元素数据管理方案

### 数据结构

`ElementData` 使用现有的 `Properties` 列表，物理属性和元属性混用同一个列表，用 key 区分：

- **元属性**（必须填）：`order`, `creativity`, `empathy`, `desire`, `mystery`, `vitality`，范围 -5 ~ 5
- **物理属性**（按需填）：如 `humidity`, `temperature`, `density`, `hardness` 等

### 批量导入

编写了编辑器脚本 `ElementDataBatchCreator.cs`，支持从 CSV 文件批量生成所有元素资产。流程：

1. 在 Excel 中填写元素数据
2. 另存为 CSV 文件（UTF-8 编码）
3. 在 Unity 菜单栏点击 `Tools → Batch Create ElementData from CSV`
4. 自动生成所有 `.asset` 文件，元属性自动填入
